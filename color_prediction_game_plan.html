Development phases
1
Foundation — auth, DB, admin shell Week 1–2
User auth
  • Register / login (JWT)
  • Phone OTP or email verify
  • Role system: player, agent, super admin
  • Session + refresh tokens
Database schema
  • users, roles, wallets
  • rounds, bets, transactions
  • markets, market_history
  • game_settings (key-value)
Wallet system
  • Coin/balance ledger table
  • Deposit + withdrawal flow
  • Idempotent debit/credit ops
  • Transaction history
Super admin shell
  • Login-protected admin panel
  • User list + balance view
  • Manual balance adjust
  • System config page
Auth APIs DB migrations Wallet APIs Admin login
2
Game engine — round loop, markets, bots Week 3–4
Round engine
  • Auto round ID generator
  • Cron-driven round lifecycle
  • State machine: init → betting → closed → result
  • Round stored in Redis (active) + DB (archive)
Market config
  • Markets table: color + number
  • Multiplier editable by super admin
  • Multiplier snapshot frozen at bet_start
  • market_history audit log
Bet processing
  • Validate: round open, balance OK
  • Atomic debit + bet insert
  • Duplicate/double-bet guard
  • Per-round bet aggregation
Bot engine
  • Bot pool: names, avatars in config
  • Random bet amount + market per bot
  • Jitter: fired 0–15s into betting window
  • Only in WS payload — zero DB writes
